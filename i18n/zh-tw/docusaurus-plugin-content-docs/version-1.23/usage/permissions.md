# 權限

Gitea 支援儲存庫的權限，以便您可以為不同的人提供不同的存取權限。首先，我們需要了解 `單元`。

## 單元

在 Gitea 中，我們稱儲存庫的子模組為 `單元`。現在我們有以下可能的單元。

| 名稱       | 描述                            | 權限      |
| ---------- | ------------------------------- | --------- |
| 程式碼       | 訪問源程式碼、文件、提交和分支。  | 讀取 寫入 |
| 問題       | 組織錯誤報告、任務和里程碑。    | 讀取 寫入 |
| 拉取請求   | 啟用拉取請求和程式碼審查。        | 讀取 寫入 |
| 發佈       | 跟蹤專案版本和下載。            | 讀取 寫入 |
| 維基       | 與合作者編寫和共享文件。        | 讀取 寫入 |
| 外部維基   | 鏈接到外部維基                  | 讀取      |
| 外部跟蹤器 | 鏈接到外部問題跟蹤器            | 讀取      |
| 專案       | 模板儲存庫的 URL                  | 讀取 寫入 |
| 包         | 與此儲存庫鏈接的包                | 讀取 寫入 |
| 操作       | 審查操作日誌或重新啟動/取消管道 | 讀取 寫入 |
| 設定       | 管理儲存庫                        | 管理      |

有了不同的權限，人們可以對這些單元執行不同的操作。

| 名稱       | 讀取                             | 寫入                    | 管理     |
| ---------- | -------------------------------- | ----------------------- | -------- |
| 程式碼       | 查看程式碼樹、文件、提交、分支等。 | 推送程式碼。              | -        |
| 問題       | 查看問題並建立新問題。           | 添加標籤、分配、關閉    | -        |
| 拉取請求   | 查看拉取請求並建立新拉取請求。   | 添加標籤、分配、關閉    | -        |
| 發佈       | 查看發佈並下載文件。             | 建立/編輯發佈           | -        |
| 維基       | 查看維基頁面。克隆維基儲存庫。     | 建立/編輯維基頁面，推送 | -        |
| 外部維基   | 鏈接到外部維基                   | -                       | -        |
| 外部跟蹤器 | 鏈接到外部問題跟蹤器             | -                       | -        |
| 專案       | 查看專案的列                     | 更改列中的問題          | -        |
| 包         | 查看包                           | 上傳/刪除包             | -        |
| 操作       | 查看操作日誌                     | 批准/取消/重新啟動      | -        |
| 設定       | -                                | -                       | 管理儲存庫 |

個人儲存庫和組織儲存庫的權限之間存在一些差異。

## 個人儲存庫

對於個人儲存庫，建立者是儲存庫的唯一所有者，對更改或刪除此儲存庫沒有任何限制。儲存庫所有者可以添加合作者來幫助維護儲存庫。合作者可以具有 `讀取`、`寫入` 和 `管理` 權限。

對於私有儲存庫，體驗類似於訪問匿名公共儲存庫。您可以訪問儲存庫中的所有可用內容，包括克隆程式碼、建立問題、回應問題評論、提交拉取請求等。如果您具有“寫入”權限，則可以推送程式碼到儲存庫的特定分支，前提是分支保護規則允許。此外，您可以更改維基頁面。具有“管理”權限，您可以修改儲存庫的設定。

但如果您不是該儲存庫的所有者，則無法刪除或轉移此儲存庫。

## 組織儲存庫

對於個人儲存庫，所有者是建立它的使用者。對於組織儲存庫，所有者是該組織所有者團隊的成員。所有權限取決於團隊權限設定。

### 所有者團隊

建立組織時將建立所有者團隊，建立者將成為所有者團隊的第一個成員。所有者團隊不能被刪除，並且至少有一名成員。

### 管理員團隊

建立團隊時，有兩種類型的團隊。一個是管理員團隊，另一個是普通團隊。可以建立管理員團隊來管理一些儲存庫，其成員可以對這些儲存庫執行任何操作。只有所有者或管理員團隊的成員可以建立新團隊。

### 普通團隊

組織中的普通團隊具有單元權限設定。它可以有成員和儲存庫範圍。

- 團隊可以訪問此組織中的所有儲存庫或特殊儲存庫。
- 團隊還可以被允許建立新儲存庫或不允許。

普通團隊可以建立以執行其權限允許的操作。一個成員可以加入多個團隊。
