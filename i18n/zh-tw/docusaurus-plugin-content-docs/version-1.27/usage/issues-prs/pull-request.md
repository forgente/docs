# 合併請求

合併請求(PR)是一種提出對儲存庫進行更改的方式。
它是一種將一個分支合併到另一個分支的請求，附帶有對所做更改的描述。
合併請求通常用作貢獻者對儲存庫貢獻程式碼的方式，儲存庫的維護者可以透過對合併請求進行審查來決定是否接受這些更改。

## 建立合併請求

要建立合併請求，您需要遵循以下步驟：

1. **Fork 儲存庫** - 如果您沒有直接對儲存庫進行更改的權限，您需要將儲存庫 fork 到您自己的帳號中。
   這將建立一個您可以對其進行更改的儲存庫副本。

2. **建立分支（可選）** - 在 fork 的儲存庫中建立一個新分支，該分支包含您要提出的更改。
   給分支取一個描述性的名稱，以指示更改的內容。

3. **進行更改** - 進行您想要的更改，提交併將其推送到 fork 的儲存庫中。

4. **建立合併請求** - 轉到原始儲存庫並轉到“合併請求”選項卡。單擊“新建合併請求”按鈕，並將您的新分支選擇為源分支。
   為您的合併請求輸入描述性標題和描述，然後單擊“建立合併請求”。

## 評審合併請求

建立合併請求後，將觸發評審流程。儲存庫的維護者將收到合併請求的通知，並可以審查所做的更改。
他們可以留下評論、請求更改或批准更改。

如果維護者請求更改，您需要在分支中進行這些更改，並將更改推送到 fork 的儲存庫中。
合併請求將自動使用新更改進行更新。

如果維護者批准更改，他們可以將合併請求合併到儲存庫中。

## 關閉合併請求

如果您不接受該合併請求，您可以關閉它。
要關閉合併請求，請轉到打開的合併請求並單擊“關閉合併請求”按鈕。這將關閉合併請求並且不會將其合併。

## 使用“Work In Progress”標記

在合併請求中使用“Work In Progress”標記可以防止合併請求被意外合併。
要將合併請求標記為“Work In Progress”，您必須在其標題中添加前綴`WIP:`或`[WIP]`（不區分大小寫）。
標記前綴可以在您的`app.ini`文件中進行設定：

```
[repository.pull-request]
WORK_IN_PROGRESS_PREFIXES=WIP:,[WIP]
```

列表的第一個值將用於 helpers 程式。
