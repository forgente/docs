# 權限

Gitea 支援對儲存庫進行權限管理，這樣您就可以為不同的人員提供不同的存取權限。首先，我們需要了解 `單元（Unit）`。

## 單元（Unit）

在 Gitea 中，我們將儲存庫的子模組稱為 `單元（Unit）`。現在我們有以下幾個單元。

| 名稱           | 描述                               | 權限      |
| -------------- | ---------------------------------- | --------- |
| 程式碼           | 訪問源程式碼、文件、提交和分支。     | 讀取 寫入 |
| 工單           | 組織缺陷報告、任務和里程碑。       | 讀取 寫入 |
| 合併請求       | 啟用合併請求和程式碼審覈。           | 讀取 寫入 |
| 發佈           | 跟蹤專案版本和下載。               | 讀取 寫入 |
| 百科           | 與協作者編寫和共享文件。           | 讀取 寫入 |
| 外部百科       | 鏈接到外部維基。                   | 讀取      |
| 外部工單跟蹤器 | 鏈接到外部工單跟蹤器。             | 讀取      |
| 專案           | 模板儲存庫的 URL。                   | 讀取 寫入 |
| 包             | 鏈接到儲存庫                         | 讀取 寫入 |
| Actions        | 審查 Actions 日誌或重啟/取消工作流 | 讀取 寫入 |
| 設定           | 管理儲存庫。                         | 管理員    |

通過不同的權限，使用者可以在這些單元上執行不同的操作。

| 名稱           | 讀取                             | 寫入                           | 管理員   |
| -------------- | -------------------------------- | ------------------------------ | -------- |
| 程式碼           | 查看程式碼樹、文件、提交、分支等。 | 推送程式碼。                     | -        |
| 工單           | 查看工單並建立新工單。           | 添加標籤、分配、關閉工單。     | -        |
| 合併請求       | 查看合併請求並建立新合併請求。   | 添加標籤、分配、關閉合併請求。 | -        |
| 發佈           | 查看發佈和下載文件。             | 建立/編輯發佈。                | -        |
| 百科           | 查看百科頁面。克隆百科儲存庫。     | 建立/編輯百科頁面，推送更改。  | -        |
| 外部百科       | 鏈接到外部百科。                 | -                              | -        |
| 外部工單跟蹤器 | 鏈接到外部工單跟蹤器。           | -                              | -        |
| 專案           | 查看面板。                       | 在面板之間移動工單。           | -        |
| 包             | 查看包                           | 上傳/刪除包                    | -        |
| Actions        | 查看 Actions 日誌                | 同意 / 取消 / 重啟             | -        |
| 設定           | -                                | -                              | 管理儲存庫 |

個人儲存庫和組織儲存庫之間的權限存在一些差異。

## 個人儲存庫

對於個人儲存庫，建立者是儲存庫的唯一所有者，對於該儲存庫的任何更改或刪除沒有限制。
儲存庫所有者可以添加協作者來幫助維護儲存庫。協作者可以擁有 `讀取（Read）`、`寫入（Write）` 和 `管理員（Admin）` 權限。

訪問私有儲存庫的體驗與訪問匿名公共儲存庫類似。您可以訪問儲存庫中的所有可用內容，包括克隆程式碼、建立工單、回覆工單評論、提交拉取請求等。如果你有 "寫"權限，只要分支保護規則允許，你就可以向儲存庫的特定分支推送程式碼。此外，你還可以修改百科頁面。有了 "管理"權限，你就可以修改儲存庫的設定。

但如果你不是該儲存庫的所有者，就不能刪除或轉移該儲存庫。

## 組織儲存庫

對於個人儲存庫，所有者是建立它的使用者。而對於組織儲存庫，所有者是該組織中的所有者團隊成員。對該組織儲存庫的所有權限都取決於團隊權限設定。

### 所有者團隊

建立組織時將自動建立所有者團隊，建立者將成為所有者團隊的第一名成員。所有者團隊不可刪除，且至少有一名成員。

### 管理員團隊

建立團隊時，有兩種類型的團隊。一種是管理員團隊，另一種是普通團隊。可以建立一個管理員團隊來管理某些版本庫，其成員可以對這些版本庫做任何事情。只有所有者或管理員團隊的成員才能建立新團隊。

### 普通團隊

組織中的普通團隊具有可以根據`單元(Unit)`進行權限設定。它可以有成員和儲存庫範圍。

- 一個團隊可以訪問所屬組織的所有儲存庫或特殊儲存庫。
- 也可以設定該團隊是否有建立新儲存庫的權限。

可以透過建立 "普通團隊"，並通過權限控制對其行為進行限制。一名成員可以加入多個團隊。
